aspectj实现静态方法织入

发表于 2015-04-15   |   分类于 aop

性能测试一直通不过去，于是想看看方法中的子方法执行所需时间。spring aop 正好合适。 在执行方法开始前记录时间，执行方法之后记录时间，两个时间求差，log输出。

想到了好方法就开始写，于是代码写出来如下：

@Aspect
@Component
public class HttpInterfaceAOP {

    @Pointcut("execution(* http.HttpInterface.*(..))")
    public void pointCut() {
    }
    @Around("pointCut()")
    public Object paround(ProceedingJoinPoint pjp) throws Throwable {
            System.out.println("around start..");
            Object o = null;
            try {
                    o = pjp.proceed();
            } catch (Throwable ex) {
                    System.out.println("error in around");
                    throw ex;
            }
            System.out.println("around end");
            return o;
    }
}

执行结果出来之后傻眼了，并没有输出时间。百死不得解。后来一看，aop拦截的全是静态方法。然后去网上搜了搜说spring aop不支持静态方法织入，也不支持final类的方法织入。springaop是基于cglib的，基于继承的方式实现的。而被final标记的类不能被继承，静态方法也不能被覆盖，因此spring aop不支持了。

这么好的想法不能利用，果断不能忍，再去网上找找看看有什么遗漏的，发现了aspectj，也是aop的一种实现，找了资料之后写出了如下的代码：

public aspect HttpInterfaceAspectJ {


    private Logger logger = Logger.getLogger(this.getClass().getName());
        pointcut pointPointCut():execution(public * http.HttpInterface.*(..));
        Object around() : pointPointCut() {
        long st= System.currentTimeMillis();
        Object obj = proceed();
        long et = System.currentTimeMillis();
        thisJoinPoint.getSignature();
        logger.info("方法-" + thisJoinPointStaticPart.getSignature().getDeclaringType().getName() + "." +
            thisJoinPointStaticPart.getSignature().getName()+"耗时:"+(et-st));
        return obj;
    }
}

其中代码的首行aspect不是class，interface等常见的java关键字，它是aspectj自己定义的语法，需要aspectj的编译器，可以从官网下载到，既有eclipse的插件，又有独立的开发包。

为了方便我选用的是eclipse的插件，找到对应eclipse版本的插件下载zip包直接安装即可。

需要注意的是要将工程转换为aspectj工程，其实就是编译器变成了asj（.project中<name>org.eclipse.jdt.core.javabuilder</name>变成了<name>org.eclipse.ajdt.core.ajbuilder</name>），buildpath中多了aspectj的jar包（.classpath中多了一条<classpathentry kind="con" path="org.eclipse.ajdt.core.ASPECTJRT_CONTAINER"/>）。asj是一个增强型的java编译器，除了实现了javac的功能还能编译aspectj程序。

写好程序之后，运行，出现了时间的日志！大功告成！

至于aspectj的实现原理，是靠的静态代理，程序运行之前class文件已经存在。springaop的动态代理需要aspectj的jar包是因为spring aop 提供aspectj风格的代理。
