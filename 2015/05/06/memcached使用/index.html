memcached使用

发表于 2015-05-06   |   分类于 memcached

安装memcached

windows下安装

windows下有安装包。安装包的地址在此。

linux下安装

linux下可以从源码安装，也可以直接使用安装包安装。
源码可以从gitbug上获取，地址在此。可以从源码安装。
在ubuntu上可以使用命令行安装。
sudo apt-get install memcached

启动memcached

打开cmd或者终端，输入memcached即可运行。通过memcached -h打印帮助信息可以看到还有更多的参数。常用的如下：
-m 指定缓存所使用的最大内存容量，单位是Megabytes，默认是64MB
-u 只有以root身份运行时才指定该参数
-d 以daemon的形式运行
-l 指定监听的地址
-p 指定监听的TCP端口号，默认是11211
-t 指定线程数，默认是4个
-c 最大同时连接数，默认是1024.
-U 指定监听的UDP端口号，默认是11211
-M 禁用LRU算法,内存耗尽时显示错误，而不是删除项
常用的格式如下： memcached -d -m 1024 -p 11211

连接memcached

memcached是一个server端的程序，官方提供了多种客户端，还提供了telnet功能。

telnet连接

在cmd或者终端中输入 telnet localhost 11211 即可连接到memcached服务器上。
在这可以输入多种命令，包括

  1. 数据存取， 如get set replace delete等
  2. 查看状态， 如stats ，stats items，stats slabs等。

客户端程序连接

java客户端有多个版本，其中xmemcached性能不错。
首先下载xmemcached的jar包，添加到工程目录。具体使用可以看官方文档。
simple-spring-memcached集成了spring和memcached客户端，依靠注解，提供了更加方便的缓存管理操作。他的代码托管在github上。
简单介绍一下simple-spring-memcached的使用吧。
将spring的jar包(版本高于3.0.7)和simple-spring-memcached的jar包放到classpath目录下，
在applicationContext.xml中配置memcached beans。
代码如下：

<aop:aspectj-autoproxy />
<import resource="simplesm-context.xml" />

<bean name="defaultMemcachedClient" class="com.google.code.ssm.CacheFactory">
    <property name="cacheClientFactory">
        <bean class="com.google.code.ssm.providers.xmemcached.MemcacheClientFactoryImpl" />
    </property>
    <property name="addressProvider">
        <bean class="com.google.code.ssm.config.DefaultAddressProvider">
            <property name="address" value="127.0.0.1:11211" />
        </bean>
    </property>
    <property name="configuration">
        <bean class="com.google.code.ssm.providers.CacheConfiguration">
            <property name="consistentHashing" value="true" />
        </bean>
    </property>
</bean>
<bean class="com.google.code.ssm.Settings">
    <property name="order" value="500" />
</bean>

这样定义beans之后，在service层可以使用@AutoWired private Cache defaultMemcachedClient获得Cache对象，进行数据的存取。
另一方面，我们可以在dao层的方法上加上注解，这样第一次从数据库里边查出来的数据会放到缓存里边，以后再请求该方法仅仅从缓存中取。
simple-spring-memcached中定义了3类注解，分别是read，invalid， update 也就是 添加 删除 更新
ReadThroughSingleCache ReadThroughMultiCache ReadThroughAssignCache
InvalidateSingleCache InvalidateMultiCache InvalidateAssignCache
UpdateSingleCache UpdateMultiCache UpdateAssignCache
项目中查询地区排名前100名的时候使用了缓存。代码如下：

@ReadThroughSingleCache(expiration=24*3600,namespace="rankResult")
public List<Map> queryRankResultArea(@ParameterValueKeyProvider Map paramMap) {
    // TODO Auto-generated method stub
    @SuppressWarnings("unchecked")
    List<Map> list = this.getSqlMapClientTemplate().queryForList("rankResultXml.queryRankResultArea",paramMap);
    return list;
}

其中expiration是缓存失效时间，默认时间是s，在此设置过期时间为1天。
namespace为命名空间，防止产生重复的key。
方法中的@ParameterValueKeyProvider注解则表示向memcached中存的key值。key值默认是通过调用Obejct.toString()方法生成的，在此调用的是map.toString()方法。
注意memcached的key不能有控制字符，比如空格，回车。请保证你的obejct.toString()不会生成这些字符。
